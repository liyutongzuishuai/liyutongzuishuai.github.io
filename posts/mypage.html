简单建站

全程「我」怎么做，你怎么复制即可。

1 装系统环境

我打开终端，先更新再装 Nginx：
bash sudo apt update && sudo apt install nginx -y sudo systemctl enable --now nginx 浏览器输服务器 IP，看到 404不好！！

1.运行sudo systemctl start nginx 结果终端回我一句：
Job for nginx.service failed because the control process exited with error code. ```

  1. 我第一时间看详细日志
    bash journalctl -u nginx.service --no-pager -n 30 里头反复提示：
    open() "/var/log/nginx/error.log" failed (2: No such file or directory) nginx: configuration file /etc/nginx/nginx.conf test failed

  2. 我立刻意识到：日志目录不存在，配置测试直接崩。
    手敲两行把坑填上：
    bash sudo mkdir -p /var/log/nginx sudo chown www-data:www-data /var/log/nginx

  3. 再次启动
    bash sudo systemctl start nginx 状态变成 active (running)，浏览器访问 IP → Welcome to nginx! 出现，我这才松口气。

我小结：
Nginx 启动失败 9 成是 日志目录缺失 或 端口 80 被占用。

先看 journalctl，再建目录/改端口/重载配置，30 秒就能复活。

2 写第一篇博客

本地新建文件 2025-10-02-hello.md，用 Markdown 写：
# Hello World - 我的第一帖

也是成功把Hello Wrold跑出来了

3 一键转成 HTML

装转换器：
bash sudo apt install markdown -y 同目录执行：
bash markdown 2025-10-02-hello.md > 2025-10-02-hello.html

4 扔到网站目录

bash sudo mkdir -p /var/www/html/posts sudo cp 2025-10-02-hello.html /var/www/html/posts/ 我顺手用ai写了个主页（可怜的前端工程师）

5 重载 & 查看

bash sudo nginx -t && sudo systemctl reload nginx 我刷新浏览器 http://IP/ → 首页出现文章链接 → 点进去就是纯 Markdown 生成的静态页。

6 以后每篇都如此

我写作流程变成：
① nvim new.md → 写内容
② markdown new.md > new.html
③ cp new.html /var/www/html/posts/
④ 在 index.html 加一行链接
⑤ reload nginx → 发布完成

「Blog 其实就是 Markdown 文件 → HTML → 丢进 Nginx 目录，写完即发布，再没别的秘密。」
